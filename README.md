# 運動飲食追蹤系統

一個基於 Windows Forms (C#) 開發的個人運動飲食追蹤系統。<br>
整合了本地資料庫與即時數據視覺化圖表，並針對使用者經驗進行深度優化，支援流暢的純鍵盤盲打操作與完善的防呆機制。
## 詳細內容介紹
### 1. 安全登入與權限管理
- **設定帳戶**：需要先建立帳戶才可以登入享有系統功能。
- **權限驗證**：拋棄傳統不安全的本地字典暫存，全面改由本地資料庫（LocalDB）安全驗證使用者憑證。
- **格式預檢**：嚴格的前端輸入驗證（帳號必須為 7 位純數字,密碼只能純數字），不符格式者由前端直接攔截，絕不無故驚動或加重資料庫負擔。

### 2. 今日狀態與個人看板
- **頁面介紹**：此頁面為系統的核心控制台，整合了個人生理指標設定、每日熱量平衡計算、以及視覺化進度條。
<img width="812" height="517" alt="image" src="https://github.com/user-attachments/assets/3464b6a2-0fb3-4997-a87e-afc61aa3af14" /><br>
### 核心技術功能：
- **動態 BMR & TDEE 醫學公式計算**：
  系統後台即時根據使用者輸入的生理數據，自動判別性別並套用標準醫學公式：
  - **男性 BMR**：$66 + (13.7 \times 體重) + (5 \times 身高) - (6.8 \times 年齡)$
  - **女性 BMR**：$655 + (9.6 \times 體重) + (1.8 \times 身高) - (4.7 \times 年齡)$
  - **TDEE 運算**：根據動態活動量係數算出每日總熱量消耗，並自動以85% 作為健康減脂的「今日建議目標攝取量」。

- **三向卡路里即時動態平衡**：
  畫面即時連動本地資料庫，透過以下公式動態計算剩餘額度，並同步渲染前端的進度條：
  $$\text{剩餘可攝取熱量} = \text{建議目標熱量} - \text{今日已攝取 } + \text{今日已消耗 }$$<br>
  若當日攝取的熱量不幸「爆卡（超過目標值）」，進度條設有防呆限制，會安全停在滿格狀態，絕不引發程式溢位或崩潰。

- **雙層歷史數據封印機制**：
  當點擊「更新個人狀態」時，系統執行雙軌寫入：
  1. 將當天數據寫入 `DailyLogs`，鎖定該日期的歷史生理指標，切換日期時能精準還原當天狀態。
  2. 同步更新 `UserProfiles` 總檔，作為未來全新日期的初始繼承數據。<br>
### 3. 飲食紀錄與營養標示 
- **頁面介紹**：此分頁負責記錄使用者當日的膳食攝取，除了基礎熱量外，更納入增肌減脂關鍵的巨量營養素（蛋白質）追蹤，並提供貼心的自動化預設輸入。
<img width="812" height="518" alt="image" src="https://github.com/user-attachments/assets/1a99f588-fc16-47d9-b1e6-d8304ef1f6e9" /><br>
### 核心技術功能：
- **常用食物字典與帶入 **：
  後台使用 `Dictionary<string, FoodInfo>` 結構建立常用食物資料庫（包含白飯、雞胸肉、茶葉蛋、無糖豆漿等）。當使用者由下拉選單選取項目時，系統會以 $O(1)$ 的極致效率在記憶體中檢索，並**自動填入**對應的熱量與蛋白質數字，免除重複手動輸入的繁瑣。
  - **優化 UX 鍵盤交互**：在食物名稱框按下 `↓` 鍵可觸發選單自動展開；選回預設提示文字時，欄位亦會自動清空，全程無縫支援純鍵盤操作。

- **自由手動輸入與前端資料預檢**：
  除了預設選單外，系統保留了 100% 的自訂彈性，允許使用者自由輸入菜單上沒有的外食。後台在執行資料庫寫入前，設有嚴格的三關防呆驗證：
  1. 使用 `string.IsNullOrEmpty` 檢查食物名稱是否為空。
  2. 套用 `int.TryParse` 強制驗證熱量必須為正整數。
  3. 套用 `double.TryParse` 驗證蛋白質格式（支援小數點），徹底在前端攔截各類非數字的盲打與亂碼，保障資料庫底層資料型態的一致性與乾淨度。

- **雙指標營養素結構化儲存**：
  不只計算卡路里，更嚴格追蹤蛋白質攝取克數。點擊「新增這餐」時，前端防呆機制會精準攔截空白或格式錯誤的輸入，確保正確後再透過底層的 SQL 命令安全儲存至 `DietLogs` 資料表中。

- **資料主鍵精準刪除**：
  下方的資料表格能即時呈現當天已吃進去的食物清單。當使用者點選某筆紀錄並點擊「刪除」時，系統會先跳出對話框進行二次確認，接著精準撈取該筆資料在資料庫中的**真正主鍵（ID）**，執行刪除命令，並重新載入並刷洗畫面。<br>

### 4. 運動紀錄與熱量消耗 
- **頁面介紹**：此分頁專門負責追蹤使用者的每日體能活動，精準記錄運動項目、持續時間與燃燒熱量，是創造「熱量赤字」與達成本地統計圖表動態渲染的核心數據來源。  
<img width="806" height="522" alt="image" src="https://github.com/user-attachments/assets/4c1bc4e5-8a0d-43d0-a140-0e1890c75030" /><br>
#### 核心技術功能：
- **動態運動耗能計量**：
  支援自由輸入各式運動型態（如：跑步、重訓、游泳等）。後台整合資料驗證機制，使用 `int.TryParse` 強制預檢「運動時間」與「消耗熱量」必須為大於零的正整數，從前端源頭杜絕各類負數或無意義字串造成的系統邏輯錯誤。

- ** 核心重整與多表狀態同步 **：
  針對 WinForms 生命週期與資料同步進行深度優化（已徹底修復空白寫入與非同步未載入 Bug）。當使用者點擊「新增運動」時，後台在單次執行緒中完美執行以下鏈式操作：
  1. 呼叫 `DatabaseHelper.AddLogItem`，將資料乾淨地寫入 `ExerciseLogs` 資料表。
  2. 自動引發 `LoadExerciseLogs()`，重新向 LocalDB 撈取當日最新陣列，並即時指派給 `dgvExercise.DataSource` 進行毫秒級的前端表格渲染。
  3. 即時聯動主分頁的 `RefreshGrids()` 與統計分頁的 `UpdateChartVisuals()`，達成全專案跨分頁數據的即時動態同步。

- **主鍵精準抹除與防護機制**：
  下方的 `dgvExercise` 資料表格具備行選取感知。當使用者選取特定運動紀錄並點擊「刪除」時，系統會精準提取當前選取物件在 SQL 中的真正主鍵 ID，執行安全抹除命令。此功能同樣掛載 `MessageBox` 二次確認防護，提供高容錯率的操作體驗。<br>
### 5. 數據視覺化與統計圖表 (Data Visualization Dashboard)
- **頁面介紹**：此分頁為系統的進階數據分析核心，將歷史資料庫中的龐雜紀錄，轉化為直觀的雙軌卡路里趨勢追蹤圖，提供使用者中長期的健康趨勢評估。
<img width="808" height="515" alt="image" src="https://github.com/user-attachments/assets/198edccb-be7e-4d7b-8ecb-e637cd72e40f" /><br>
### 核心技術功能：
- **獨立子查詢多表時間軸加總**：
  底層資料庫採用高度優化的關聯式 SQL 查詢。為了避免「飲食紀錄」與「運動紀錄」在一對多Join時產生資料筆數乘積錯亂的經典Bug，後台在 `DatabaseHelper.GetChartData` 中，使用了**兩組獨立的子查詢**分別在各自的資料表中依日期執行 `SUM(Calories)` 與 `SUM(BurnedCalories)` 聚合運算，最後再以 `LEFT JOIN` 將兩者無縫對齊於相同時間軸上，保障了統計數據的絕對精準度。

- **多動態時間維度切換**：
  前端整合下拉選單，提供「最近一週 (7天趨勢)」、「最近一個月 (30天趨勢)」與「最近一年 (按月平均)」三種動態數據維度。當使用者切換選項時，後台會即時向資料庫發送帶有不同時間條件的 SQL 指令，達成非同步的即時圖表重新抽換。

- **OxyPlot 雙 Canvas 物件獨立渲染技術**：
 整合 **OxyPlot.WindowsForms** 高階圖表引擎，實作雙 Panel 獨立渲染架構：
  - 🔵 **左圖表：每日運動消耗卡路里 **：採用 `OxyColors.DodgerBlue` 實心折線，動態綁定每日運動總燃燒值，搭配圓形標記點（MarkerType.Circle）強化 scannability。
  - 🔴 **右圖表：每日飲食攝取熱量 **：採用 `OxyColors.Red` 鮮紅折線，即時呈現每日膳食總熱量進帳。

### 6. 深度 UX 鍵盤友善優化
為了提供極致的盲打流暢度，系統擺脫了對滑鼠的依賴：
- **開機焦點鎖定**：登入畫面開啟時，游標自動聚焦於帳號輸入框。
- **Enter 鍵走位**：帳號輸入完按 `Enter` 自動跳轉密碼框；密碼輸入完按 `Enter` 秒觸發登入。
- **TabIndex 流暢穿梭**：全視窗控制項經嚴格排序，使用 `Tab` 鍵即可完美由上至下變更焦點。

### 7. 系統防護與防呆機制
- **Copy-Paste Bug 修復**：徹底解決複製貼上引發的欄位錯亂與空白資料重複寫入資料庫的邏輯漏洞。
- **二次確認機制**：登入畫面與主視窗皆掛載 `FormClosing` 事件，點擊右上角叉叉時會跳出對話框進行二次確認，防止使用者因手殘或誤觸導致編輯中的資料遺失。
